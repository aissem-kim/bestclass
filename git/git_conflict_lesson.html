수업 개요 & 시나리오

오늘의 시나리오

팀 프로젝트로 계산기 프로그램(calculator.py)을 개발 중입니다.
직원 A와 직원 B가 같은 파일의 같은 줄을 동시에 수정해서 충돌이 발생합니다.
이 충돌을 직접 해결하고 merge까지 완료하는 실습을 진행합니다.

충돌은 같은 파일의 같은 줄을 두 사람이 다르게 수정할 때 발생합니다.
오늘 실습은 이 상황을 의도적으로 만들고 → 직접 해결하는 것이 목표입니다.

환경 구성

초기 파일 내용 (calculator.py)

두 직원이 공유할 시작 파일입니다. main 브랜치에 먼저 올린 후 각자 브랜치를 만듭니다.

직원 A — 저장소 초기화
GitHub에 원격 저장소 생성 후 진행
  • ① 폴더 생성 및 초기화
    (※) 기존에 사용하던 team_project 폴더와 깃허브 team_project을 이용한다면 아래 ① 코드는 안해도 됩니다. 
    mkdir git-conflict-lab
cd git-conflict-lab
git init
git remote add origin [원격저장소URL]
  • ② calculator.py 파일 생성
    def calculate(a, b):
    result = a + b   # ← 충돌 포인트
    return result

print(calculate(3, 4))
  • ③ 첫 번째 커밋 & 푸시
    git add calculator.py
git commit -m "초기 계산기 파일 추가"
git push -u origin main
  • ④ 작업 브랜치 생성
    git checkout -b feature/add

    A는 feature/add 브랜치에서 작업합니다

직원 B — 저장소 클론
A의 저장소 주소를 받아서 진행
  • ① A의 저장소 클론
    (※) 기존에 사용하던 team_project 폴더와 깃허브 team_project을 이용한다면 아래 코드는 안해도 됩니다. 
    git clone [A의저장소URL]
cd git-conflict-lab
    기본 프로젝트에서 작업할 때는 A의 저장소 PULL 작업 
    git pull origin main

    A가 push한 파일이 그대로 받아집니다

  • ② 파일 내용 확인
    cat 명령어는 리눅스명령어입니다. 윈도우에서는 에디터로 확인할 수 있습니다. 
    cat calculator.py
git log --oneline
  • ③ 작업 브랜치 생성
    git checkout -b feature/multiply
  • ④ 준비 완료 확인
    git branch
git status

여기까지 완료되면 두 직원 모두 같은 파일을 가지고 각자의 브랜치에 있는 상태입니다.
다음 단계에서 의도적으로 같은 줄을 다르게 수정해 충돌을 만듭니다.

충돌 발생시키기

충돌 시나리오

A와 B가 calculator.py 4번째 줄 같은 위치를 각자 다르게 수정합니다.
A는 덧셈으로, B는 곱셈으로 바꿉니다. → 나중에 merge하면 충돌!

직원 A — 덧셈 기능으로 수정
  • ① 파일 수정 (4번째 줄 주석 추가)
    result = a + b  # 덧셈 기능 추가
  • ② 커밋 & 푸시
    git add calculator.py
git commit -m "덧셈 기능 구현"
git push origin feature/add
  • ③ main에 merge
    git checkout main
git merge feature/add
git push origin main

    ✅ A의 코드가 먼저 main에 반영됩니다

직원 B — 곱셈 기능으로 수정
  • ① 파일 수정 (같은 4번째 줄을 다르게!)
    result = a * b  # 곱셈 기능으로 변경

    ⚠ A와 같은 줄을 다르게 수정!

  • ② 커밋 & 푸시
    git add calculator.py
git commit -m "곱셈 기능으로 변경"
git push origin feature/multiply
  • ③ main 최신 상태 가져오기
    git checkout main
git pull origin main
  • ④ merge 시도 → 충돌 발생!
    git merge feature/multiply

    ❌ CONFLICT 메시지가 뜹니다!

충돌 파일 모습 (calculator.py)
충돌 마커가 포함된 파일 내용
def calculate(a, b):
<<<<<<< HEAD   ← main의 코드 (A)
result = a + b # 덧셈 기능 추가
=======   ← 구분선
result = a * b # 곱셈 기능으로 변경
>>>>>>> feature/multiply   ← B의 코드
return result
print(calculate(3, 4))
마커의미
<<<<<<< HEAD ~ =======현재 브랜치(HEAD)의 코드 — A가 작성
======= ~ >>>>>>>합치려는 브랜치의 코드 — B가 작성

마커 3개(<<< === >>>)를 모두 삭제하고 원하는 코드만 남겨야 해결됩니다.

충돌 해결

해결 방법 3가지

상황에 따라 아래 중 하나를 선택합니다. 오늘 실습은 방법 ③ 협의 후 직접 수정으로 진행합니다.

①
내 코드(HEAD) 유지 — A의 코드만 남기기
git checkout --ours calculator.py

마커를 포함해 B의 코드를 모두 버리고 A 코드만 남깁니다

②
상대방 코드 채택 — B의 코드만 남기기
git checkout --theirs calculator.py

A의 코드를 버리고 B 코드만 남깁니다

③
협의 후 직접 수정 (권장) — 두 기능 모두 살리기
# 파일을 편집기로 열어 마커 삭제 후 아래처럼 수정
def calculate(a, b, op="add"):
    if op == "add":
        result = a + b   # 덧셈
    else:
        result = a * b   # 곱셈
    return result

✅ 마커 3개(<<< === >>>)를 반드시 모두 삭제해야 합니다!

(직원 B — merge 담당)
충돌 해결 완료 명령어 순서
  • ① 충돌 상태 확인
    git status

    both modified: calculator.py 가 표시됩니다

  • ② 편집기로 파일 열어 마커 삭제 및 코드 정리
    code calculator.py      # VS Code
# 또는
notepad calculator.py   # 메모장
  • ③ 해결된 파일 스테이징
    git add calculator.py
  • ④ merge 커밋 완료
    git commit -m "충돌 해결: 덧셈/곱셈 기능 통합"
  • ⑤ 원격 저장소에 반영
    git push origin main
  • ⑥ 로그로 결과 확인
    git log --oneline --graph --all

    ✅ 두 브랜치가 합쳐진 merge commit이 보이면 성공!

Q1. 충돌이 발생하는 조건은 무엇인가요?

같은 파일의 같은 줄(혹은 근처)을 두 브랜치에서 서로 다르게 수정한 후 merge할 때 발생합니다.

Q2. 충돌 파일에서 ======= 아래쪽 코드는 어느 브랜치의 것인가요?

merge 하려는 브랜치(feature 브랜치)의 코드입니다. 위쪽은 HEAD(현재 브랜치)의 코드입니다.

Q3. 충돌을 해결한 후 반드시 실행해야 하는 명령어 두 가지는?

git add [파일명] 으로 스테이징한 후, git commit 으로 merge 커밋을 완료해야 합니다.

Q4. 내 코드를 모두 버리고 상대방 코드만 채택하는 명령어는?

git checkout --theirs [파일명] 입니다. 반대로 내 코드를 유지하려면 --ours 를 사용합니다.

Q5. 충돌 해결 후 두 브랜치 히스토리를 시각적으로 확인하는 명령어는?

git log --oneline --graph --all 명령어로 브랜치 분기와 merge 지점을 그래프로 확인할 수 있습니다.
